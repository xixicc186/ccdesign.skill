03 / process

thinks before it draws.

8 steps · brief-first · no slop
step 01

identify the output type

Slides? Poster? Living prototype? PDF report? Video? Different deliverables, different physics. The skill names the output before doing anything else, so you can correct course in one sentence.

pptxhtmlpdfimageremotion
step 02

ask, then shut up

One numbered list of questions — audience, brand assets, style reference. No design happens until you reply. Restraint is the discipline.

audiencebrandreference
step 03

read the brand's own design.md

Pick a style from the local design-md library — exact hex codes, type pairings, do-and-don't lists. The agent doesn't approximate Stripe purple. It uses the value Stripe uses.

38 systemslocal firstweb fallback
step 04

write a design brief

Before any code: a single creative thesis, the "wow moment," the visual language and the interaction signature. You read it. You correct. Only then does the agent open its editor.

thesishighlightsignature
step 05

map every effect to a real implementation

An interaction blueprint: each component → which library → which technique → which local reference file. No invented APIs, no hallucinated CSS.

gsapscrollrevealanime.jsvanilla 3d-tilt
step 06–09

generate, verify, hand off

Three variants by default — conservative, standard, bold. Screenshots at full resolution. Sub-skills (pptx / pdf / remotion) take it home.

3 variantsscreenshot QAhandoff
